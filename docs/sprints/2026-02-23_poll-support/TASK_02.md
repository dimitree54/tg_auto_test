## Description

Teach the serverless testing infrastructure to handle `sendPoll` API calls. This spans four layers: the stub request handler, the response processor routing, the message factory, and the `ServerlessMessage` model.

### 1. StubTelegramRequest ÔÇö handle `sendPoll`

Add `"sendPoll": self._handle_send_poll` to `_handlers` in `stub_request.py`.

Implement `_handle_send_poll` (likely in `stub_request_media.py` as a new method on `MediaMixin`, or in a new `stub_request_poll.py` mixin ÔÇö choose based on file size limits).

The handler must:
- Parse `question`, `options` (JSON array of strings), `is_anonymous`, `type` from `parameters`.
- Build a Telegram API response message containing a `poll` object:
  ```json
  {
    "id": "<generated_poll_id>",
    "question": "What's your favorite color?",
    "options": [
      {"text": "Red", "voter_count": 0},
      {"text": "Green", "voter_count": 0},
      {"text": "Blue", "voter_count": 0}
    ],
    "total_voter_count": 0,
    "is_closed": false,
    "is_anonymous": false,
    "type": "regular",
    "allows_multiple_answers": false
  }
  ```
- Generate a unique `poll_id` string (e.g., incrementing counter like `"poll_1"`).
- Store the poll-to-message mapping internally so that `process_poll_answer` (TASK_03) can look up which poll belongs to which message.

### 2. Response processor ÔÇö route `sendPoll`

Add `"sendPoll"` to `_MESSAGE_METHODS` in `response_processor.py`.

### 3. Message factory ÔÇö build poll `ServerlessMessage`

Add a `_build_poll_message` function in `message_factory.py` (or a new `message_factory_poll.py` if file size is tight).

It must:
- Extract `poll` data from `call.result` (the API response stored on `TelegramApiCall`).
- Build a `ServerlessMessage` with the new `poll_data` field populated.

Register `"sendPoll"` in `_METHOD_BUILDERS` if using the builder dispatch, or handle it as a special case in `build_serverless_message` (like `sendInvoice` is handled).

### 4. ServerlessMessage ÔÇö add `poll` property

Add a `poll_data` field (or similar) to `ServerlessMessage` in `models.py`.

Add a `poll` property that returns a Telethon `MessageMediaPoll` built from the stored data:
```python
from telethon.tl.types import MessageMediaPoll, Poll, PollAnswer

@property
def poll(self) -> MessageMediaPoll | None:
    if self.poll_data is None:
        return None
    answers = [
        PollAnswer(text=opt["text"], option=bytes([i]))
        for i, opt in enumerate(self.poll_data["options"])
    ]
    tl_poll = Poll(
        id=int(self.poll_data["id"]),
        question=self.poll_data["question"],
        answers=answers,
    )
    return MessageMediaPoll(poll=tl_poll, results=...)
```

Note: Check the exact Telethon TL type signatures ÔÇö `PollAnswer.text` may need a `TextWithEntities` wrapper in newer Telethon versions. Verify at implementation time.
